# 2026-04-02 AI Coder Log: Fork COW 实现

## 1. 任务目标 (Objective)
- 在 PulseOS 上实现可工作的 fork COW（Copy-On-Write），替换原先深拷贝 fork 导致的内存压力问题。
- 补齐 `stat` 对目录路径的处理，避免目录场景被错误返回 `EISDIR`。

## 3. 详细修改 (Detailed Changes)
- **COW 核心（axmm 侧）**:
  - 在 `alloc` backend 增加共享页引用计数表（基于 `SpinNoIrq<BTreeMap<usize, usize>>`）。
  - 新增共享页引用管理逻辑：
    - `cow_inc_frame_ref(frame)`：fork 共享时递增引用。
    - 释放页时先减引用，只有引用归零才真正 `dealloc_pages`。
  - 扩展缺页处理逻辑：
    - 若 fault 是写访问，且当前 PTE 无写权限，但区域原始权限可写，则执行 COW 分离：
      1. 分配新页
      2. 拷贝旧页内容
      3. `remap` 到新页并恢复写权限
      4. 降低旧页引用
    - 兼容 `populate=true/false` 的映射类型。

- **地址空间接口补充（axmm aspace）**:
  - 新增 `AddrSpace::remap_page(vaddr, paddr, flags)`，用于 fork 时将子页映射到父页物理帧。

- **fork 路径改造（PulseCore）**:
  - `spawn_fork_from_trap_frame` 从深拷贝改为 COW 共享映射：
    - 扫描父进程当前已映射用户页。
    - 子进程侧建立同物理页映射。
    - 对可写用户页：父子初始都改为只读，等待写缺页触发 COW。
    - 所有共享页增加引用计数，避免一方提前释放导致另一方悬挂。
  - 保留 `heap_top` 上界限制，避免无意义遍历超大地址范围。

- **sys_clone 语义修复（Pulse Syscalls）**:
  - 子进程 trap frame 的 `sepc` 增加 4（RISC-V ecall 指令长度），确保 child 从 syscall 返回点继续执行，而不是重复陷入 `clone`。

- **页错误处理修复（PulseCore Trap）**:
  - page fault 转发时保留完整访问标志，不再只传 `WRITE`，避免合法缺页被误判。

- **ls 目录场景修复（POSIX FS）**:
  - `sys_stat` 改为同时支持文件与目录：
    - 文件路径走 `File::stat()`。
    - 目录路径走 `DirFile::stat()`。
  - 修复此前 `sys_stat => Err(EISDIR)` 干扰 `ls` 行为的问题。
