## 2025-05-20 - Avoid format! macro allocations in hot loops for prefix checking
**Learning:** Checking string prefixes with dynamically constructed strings via `format!` inside hot loops causes excessive heap allocations. In this codebase's unikernel environment, memory allocation overhead scales with loop iterations, severely degrading registry or structure traversal performance.
**Action:** When validating target string matches, use idiomatic zero-allocation methods like `strip_prefix(pattern)` followed by slicing or `starts_with` rather than creating temporary `String` instances with `format!`.
